## Опис

У лабораторній роботі розширено програму з Lab 3.  
Додано підтримку параметру, який дозволяє виводити **усі стани поля на кожному кроці падіння фігури**.  
При цьому початковий режим роботи (коли друкується тільки фінальний стан) збережено.

Також у цьому завданні потрібно:

- використати бібліотеку для створення mock-об’єктів (test doubles);
- замінити власні mock’и з Lab 3 на mock’и із вибраної бібліотеки;
- зберегти архітектурний поділ на три шари:
  - логічний шар,
  - введення/виведення (I/O),
  - комунікаційний шар із DI (dependency injection);
- покрити тестами усі шари;
- налаштувати GitHub Actions;
- оформити PR із робочим рішенням та PR із тестами, що падають.

## Вимоги

* Python 3.11+
* `pip`
* `pytest`

## Налаштування середовища / встановлення залежностей

```bash
pip install -r requirements.txt
```

```bash
python -m pytest
```

## Запуск програми вручну

Вхідний файл `input.txt` (приклад):

```bash
7 8
..p.....
.ppp....
..p.....
........
...#....
...#...#
#..#####
```

### Windows (для вiдображення всiх крокiв)

```bash
python tetis.py input.txt --all
```
або

```bash
./tetis input.txt --all
```

### Linux (для вiдображення всiх крокiв)

```bash
python3 tetis.py input.txt --all
```
або

```bash
./tetis input.txt --all
```

для запуску фiнального стану використовувати тi самi команди без `--all`
